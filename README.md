### 🔭 What I'm building currently

- 🔬 **[meta-research](https://github.com/theohsiung/metaResearch)** — my main focus. **AI for Science**: a general-purpose autonomous design-optimization framework where a coding agent runs a research loop in-session — propose a design → score against a simulator → read diagnostics (e.g. heatmaps) → form the next falsifiable hypothesis → repeat, committing every result to git as append-only experience. Built on the shoulders of [karpathy/autoresearch](https://github.com/karpathy/autoresearch) (ergonomics) and [stanford-iris-lab/meta-harness](https://github.com/stanford-iris-lab/meta-harness) (methodology) — I'm honored to build on top of their work and contribute back to the AI-for-Science community.
- 🧭 Contributing to **[vLLM Semantic Router](https://github.com/vllm-project/semantic-router)** — system-level intelligent routing for mixture-of-models across cloud, data center & edge. I work across the request lifecycle (**signal → decision → algorithm**): session token budgets, model-switch gating, and cache-retention directives. It's a privilege to contribute to a project shaping how the open-source LLM-serving stack will route inference at scale.
- 🤖 Building **agentic systems & developer tooling** — coding agents, agent skills, and harnesses for LLM-driven scientific workflows.
- 📊 Exploring **LLM evaluation & reasoning** — long-context retrieval (needle-in-a-haystack) and ARC-AGI-style reasoning.

### 🧠 Background

From **computer-vision research** (large-pose face reenactment, FLAME 3D face models, diffusion avatars) to **LLM systems & inference infrastructure**, and now toward **AI for Science** — letting autonomous agents drive the research loop itself. I like working close to the model: simulators, routing, serving, evaluation, and the agent layer on top.
